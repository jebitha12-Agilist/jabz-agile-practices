# Wind Turbine Configurator – Definition of Done (DoD) ✅

## 🌀 Sprint DoD
A user story or feature is considered **done** within a sprint when:

- **DEV (Development)** → Code implemented, peer‑reviewed, and merged into sprint branch.  
- **UT (Unit Tests)** → Automated unit tests written and passed.  
- **INT (Integration)** → ERP/SCADA connectors validated in staging.  
- **SIM (Simulation Accuracy)** → Aerodynamic/load simulations ≥95% accuracy.  
- **UI (User Interface)** → Configurator UI updated and accessible.  
- **DOC (Documentation)** → Updated in Confluence/Jira for operators and engineers.  
- **QA (Quality Assurance)** → Smoke and regression tests passed with zero critical defects.  
- **AC (Acceptance Criteria)** → All story acceptance criteria met and signed off by Product Owner.  

---

## 🚀 Release DoD
A release is considered **done** when:

- **REL (Release Branch)** → Code merged into release branch, tagged, and versioned.  
- **CI/CD (Continuous Integration/Continuous Deployment)** → Pipeline executed successfully with automated builds and deployments.  
- **PERF (Performance Testing)** → Configurator response time validated under 100+ concurrent users.  
- **UAT (User Acceptance Testing)** → Signed off by field engineers and OEM partners.  
- **COMP (Compliance)** → IEC 61400 and local energy authority checks passed.  
- **SEC (Security Validation)** → Vulnerability scans completed, no critical findings.  
- **RBK (Rollback Plan)** → Tested and ready in case of failure.  
- **DOC (Release Documentation)** → Release notes, operator manuals, and stakeholder communication updated.  
- **MET (Metrics)** → Deployment success rate, change failure rate, and stakeholder satisfaction tracked.  
- **RETRO (Retrospective)** → Release retrospective scheduled to capture lessons learned.  

---

## 🌟 Key Takeaway
- **Sprint DoD** ensures each story is production‑ready at the team level.  
- **Release DoD** ensures the entire product increment is safe, compliant, and stakeholder‑approved for deployment.
